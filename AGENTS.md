# AGENTS.md

## 1. 프로젝트 개요

프로젝트 피클은 로컬 디렉토리의 동영상 파일을 스캔, 재생, 평가, 정리하는 Windows 데스크톱 앱이다.

목표는 다음과 같다.
- 동영상 파일을 DB에 적재하고 관리한다.
- 플레이어 중심 UI에서 빠르게 검토한다.
- 태그, 설명, 썸네일, 스냅샷, 파일명 변경 기능을 제공한다.

---

## 2. 사용 기술

- Qt 6
- Qt Quick / QML
- C++20
- CMake
- Qt Multimedia
- Qt SQL (SQLite)
- Windows first

예정 보조 도구:

## 3. 최우선 아키텍처 규칙

### 3.1 UI와 로직 분리
- QML은 UI만 담당한다.
- 파일 시스템 접근, DB 처리, 스캔, 해시 계산, 메타데이터 추출, 이름 변경 로직은 C++에서 처리한다.
- 비즈니스 규칙을 QML에 넣지 않는다.

### 3.2 모델 중심 리스트 구현
- 우측 라이브러리 리스트는 `QAbstractListModel` 기반으로 구현한다.
- delegate 내부에 영구 상태를 저장하지 않는다.
- 선택 상태, 평점, 검토 상태 등은 모델/백엔드에 둔다.

### 3.3 데이터 저장 규칙
- 영속 데이터는 SQLite에 저장한다.
- DB 스키마 변경 시 migration을 추가한다.
- 스캔 결과는 upsert 가능한 구조를 우선한다.

### 3.4 기능 구현 원칙
- 초기에는 기본 제공 Qt Quick Controls를 우선 사용한다.
- 꼭 필요하지 않으면 복잡한 커스텀 컨트롤을 만들지 않는다.
- 초기 버전에서는 실시간 감시보다 수동 재스캔을 우선한다.

## 4. 작업 방식

### 4.1 마일스톤 단위 작업
- 한 번에 하나의 작은 마일스톤만 구현한다.
- 항상 앱이 빌드 가능한 상태를 유지한다.
- 거대한 한 번의 변경보다 작은 단계적 변경을 선호한다.

### 4.2 허용되는 변경
- 현재 마일스톤 범위 내의 파일 추가/수정
- 필요한 클래스/모듈/문서 추가
- CMake 정리
- 테스트 코드 추가
- 문서 갱신

### 4.3 피해야 할 변경
- 범위를 벗어난 대규모 리팩터링
- UI 전체 갈아엎기
- 임의의 기술 스택 변경
- 파괴적인 Git 명령 사용
- 근거 없는 아키텍처 재설계

## 5. 코딩 규칙

### C++
- 가능한 한 책임이 명확한 작은 클래스로 분리한다.
- `Controller`, `Service`, `Model` 역할을 구분한다.
- 장시간 작업은 UI 스레드를 막지 않도록 설계한다.
- 하드코딩된 경로를 피한다.
- 오류는 조용히 무시하지 말고 로그나 상태로 드러낸다.

### QML
- 화면 표현과 사용자 상호작용에 집중한다.
- 복잡한 로직, 파일 시스템 처리, DB 접근은 넣지 않는다.
- 재사용 가능한 컴포넌트는 `qml/components`에 둔다.
- 페이지 단위 구성은 `qml/pages`에 둔다.

### DB
- 테이블 생성은 migration 경로를 따른다.
- 컬럼 추가/수정 시 기존 데이터 보존을 우선한다.
- 초기 핵심 테이블: `media_files`, `tags`, `media_tags`, `snapshots`, `scan_roots`, `app_settings`

## 8. 빌드 및 검증 원칙

- 가능한 범위에서 빌드 확인을 수행한다.
- 실행 가능한 경우 앱 실행 여부를 확인한다.

## 9. 금지 사항
