# Roadmap

## 원칙

- 항상 빌드 가능한 상태를 유지한다.
- 한 번에 하나의 작은 마일스톤만 구현한다.
- 각 단계는 명확한 완료 기준을 가진다.
- 실제 구현은 Codex가 맡더라도, 각 단계의 범위와 완료 여부는 사람이 통제한다.

---

## M00. 저장소 부트스트랩

### 목표
Qt 기반 프로젝트의 최소 실행 환경과 문서 구조를 만든다.

### 범위
- Qt Quick Application 생성
- CMake 기반 프로젝트 구성
- 기본 `src`, `qml`, `docs`, `assets`, `tests` 디렉토리 생성
- `README.md`, `AGENTS.md`, `docs/architecture.md`, `docs/roadmap.md` 추가

### 완료 기준
- 앱이 빌드된다.
- 기본 창이 실행된다.
- 문서 4개가 저장소에 존재한다.

### 제외
- 실제 기능 구현

---

## M01. UI 쉘

### 목표
실제 기능 없이 프로젝트 피클의 화면 구조를 먼저 고정한다.

### 범위
- 중앙 플레이어 영역 placeholder
- 하단 재생바 placeholder
- 우측 상단 라이브러리 리스트 UI
- 우측 하단 상세 정보 패널 UI
- 목업 데이터 10개 표시

### 완료 기준
- 앱이 실행된다.
- 메인 레이아웃이 보인다.
- 리스트 선택 시 우측 하단 정보가 바뀐다.

### 제외
- 실제 DB
- 실제 스캔
- 실제 동영상 재생

---

## M02. DB 초기화

### 목표
SQLite DB를 만들고 기본 스키마를 준비한다.

### 범위
- DB 파일 생성
- migration 테이블 생성
- `media_files`, `tags`, `media_tags`, `snapshots`, `scan_roots`, `app_settings` 생성
- 앱 시작 시 DB 초기화 루틴 작성

### 완료 기준
- 앱 실행 시 DB가 생성된다.
- 테이블이 자동 생성된다.
- 재실행 시 기존 DB를 재사용한다.

### 제외
- 실제 스캔 반영

---

## M03. 디렉토리 스캔

### 목표
사용자가 선택한 폴더에서 동영상 파일을 찾아 DB에 적재한다.

### 범위
- 폴더 선택 기능
- 재귀 스캔
- 기본 파일 정보 수집
- 확장자 기반 동영상 후보 필터링
- DB upsert

### 완료 기준
- 루트 폴더를 선택할 수 있다.
- 동영상 파일이 DB에 들어간다.
- 중복 경로는 update 된다.
- 스캔 결과가 리스트에 반영된다.

### 제외
- 상세 메타데이터
- 실시간 감시

---

## M04. 라이브러리 리스트 연결

### 목표
DB의 실제 데이터를 우측 리스트에 연결한다.

### 범위
- `QAbstractListModel` 구현
- 정렬(이름/크기/수정일)
- 검색(파일명/설명)
- 썸네일 표시/비표시 토글
- 선택 상태 연결

### 완료 기준
- DB 데이터가 리스트에 보인다.
- 정렬이 동작한다.
- 검색이 동작한다.
- 선택된 파일의 상세정보가 우측 하단에 반영된다.

### 제외
- 태그 필터 고도화
- 대표 썸네일 생성

---

## M05. 기본 재생 기능

### 목표
선택한 동영상을 중앙 플레이어에서 재생한다.

### 범위
- `MediaPlayer` 연동
- `VideoOutput` 연결
- 재생/일시정지
- 시크바
- 볼륨
- 현재 시간 표시

### 완료 기준
- 리스트 선택 시 동영상이 재생된다.
- 재생/일시정지가 동작한다.
- 시크가 가능하다.
- 재생 상태가 UI와 동기화된다.

### 제외
- 배속
- 자막
- 마지막 위치 저장

---

## M06. 상세 메타데이터 및 편집

### 목표
선택 파일의 상세 정보를 표시하고 설명/태그/상태를 저장한다.

### 범위
- ffprobe 기반 메타데이터 추출
- 해상도/길이/비트레이트/코덱 표시
- description 편집
- 태그 추가/삭제
- review status, rating 저장

### 완료 기준
- 선택 파일의 상세 정보가 표시된다.
- 설명과 태그를 수정할 수 있다.
- 수정 사항이 DB에 저장된다.

### 제외
- 태그 자동 추천
- 고급 필터 조합 UI

---

## M07. 파일 관리 기능

### 목표
영상 관리 도구로서 핵심 편집 기능을 추가한다.

### 범위
- F2 이름 바꾸기
- 검토 상태 변경
- 즐겨찾기/삭제후보 처리
- 마지막 재생 위치 저장

### 완료 기준
- F2 또는 액션으로 파일명을 바꿀 수 있다.
- 상태 변경이 리스트와 상세 패널에 반영된다.
- 마지막 재생 위치가 저장된다.

### 제외
- 실제 삭제 수행
- 휴지통 통합

---

## M08. 스냅샷 및 썸네일

### 목표
썸네일/스냅샷 기능을 추가해 정리 효율을 높인다.

### 범위
- 특정 시점 스냅샷 생성
- 대표 썸네일 지정
- 썸네일 경로 저장
- 리스트에서 썸네일 표시

### 완료 기준
- 스냅샷이 생성된다.
- 대표 썸네일을 저장할 수 있다.
- 리스트 썸네일 토글이 동작한다.

### 제외
- 대량 썸네일 일괄 생성 최적화

---

## M09. 품질 보강

### 목표
실사용 가능한 수준으로 안정성을 높인다.

### 범위
- 예외 처리 보강
- 에러 메시지 정리
- 스캔 취소/진행률 개선
- DB migration 정리
- 로그 정리
- UI 미세 개선

### 완료 기준
- 주요 실패 상황에서 앱이 비정상 종료하지 않는다.
- 스캔/재생/편집 흐름이 안정적이다.
- 주요 기능 수동 점검 체크리스트를 통과한다.

1. 변경한 파일 목록
2. 구현한 내용
3. 확인한 내용
4. 남은 문제
5. 다음 작업 제안
